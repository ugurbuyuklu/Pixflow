# UI Specification

> Wireframes and component specifications for Borgflow web interface.

## Design Principles

- **Minimal** - Focus on functionality, no unnecessary decoration
- **Dark mode** - Default theme (easier on eyes for creative work)
- **Efficient** - Minimum clicks to accomplish tasks
- **Clear feedback** - Always show what's happening (loading, progress, errors)

## Components

### ConceptInput

```tsx
interface ConceptInputProps {
  value: string;
  onChange: (value: string) => void;
  disabled?: boolean;
}
```

### PromptSlider

```tsx
interface PromptSliderProps {
  value: number;        // 1-10
  onChange: (value: number) => void;
  disabled?: boolean;
}
```

### PromptList

```tsx
interface PromptListProps {
  prompts: Prompt[];
  selectedIds: string[];
  onSelect: (id: string) => void;
  onSelectAll: () => void;
  onClearSelection: () => void;
  activePromptId: string;
  onSetActive: (id: string) => void;
}
```

### PromptPreview

```tsx
interface PromptPreviewProps {
  prompt: Prompt;
  onCopy: () => void;
  onEdit: () => void;
  onRegenerate: () => void;
  onDelete: () => void;
}
```

### ProgressCard

```tsx
interface ProgressCardProps {
  status: 'queued' | 'generating' | 'done' | 'error';
  progress?: number;    // 0-100 for generating
  thumbnail?: string;   // Base64 or URL for done
  filename: string;
  onView?: () => void;
  onRetry?: () => void;
}
```

### ImageDropzone

```tsx
interface ImageDropzoneProps {
  image?: File;
  onDrop: (file: File) => void;
  onClear: () => void;
}
```
